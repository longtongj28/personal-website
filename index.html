Grades? Web App

  • Full stack application implemented using the MongooDB, Express, React, and Node.js tech stack.
  • Designed and managed front-end application state through several components (React)
  • Designed and implemented the REST API with the CRUD operations that allowed communication with MongoDB, my database provider (Node.js, Express, and Postman)
  • The structural model for the data was this:
    User->Courses->Course Categories->Category Assignments
  • I had several challenges with the data that were fixed by carrying the item key/ID from higher levels of data to lower levels, which helped me implement delete and put requests.
  • I needed a provider to deploy my website, since it's not a typical static website. I chose Heroku for that, and learned how to integrate the necessary heroku scripts into my own applications build script.
  • Reflections:
    I would use React Redux for global application state rather than managing and passing through components.
    It also would have been a better idea to design the entire structural model for the data before implementing it.
  • Check it out here!

MATLAB Battleship Game

  • Meet MATLAB Battleship, a terminal game that was deceptively difficult to make.
  • User input verification was the biggest challenge. My group members helped me by testing the game continuously and reporting errors to me.
  • Non-valid inputs were taken into consideration in designing the game.
  • For example, placing ships on top of each other or outside the board prompts the user again.
  • The next most challenging part was the single player AI. There are two difficulties here, easy and hard.
  • The easy difficulty simply guesses random areas on the board. I had a lot of fun figuring out what the hard difficulty meant.
    Eventually, I made a simple algorithm for the computer to keep guessing until it hits a ship. If it does, then search around that hit.
  • Download MATLAB's IDE and play Battleship after downloading it from here!
